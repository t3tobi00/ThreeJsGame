## Decisions & Reasoning
- **ResourceStack works in both world and local space** — the key insight is that `items[i].position` is always relative to whatever parent it was added to (scene or group), so the same update loop works for both. Villager meat uses group-local base `(0, 1.2, -0.25)`.
- **Caller owns the mesh** — ResourceStack and ResourceTransfer never touch the scene. They only animate positions. This keeps responsibilities clean and prevents double-dispose bugs.
- **`this.stack = this._resourceStack.items` (reference, not copy)** — SellingSystem reads `.stack.length` externally; by making `stack` point to the same array object, no SellingSystem changes needed.
- **`advanceQueue()` immediately on `setApproachingTable()`** — the old code advanced queue only after the exiting timeout, leaving a gap where no one was in queue position 0, causing nextInQueue to return null permanently.
